RSI calculation requirements:
Implement RSI using Wilder’s original smoothing method, not a simple rolling average after initialization.

Algorithm:
- Use close prices by default.
- delta = current_close - previous_close
- gain = max(delta, 0)
- loss = abs(min(delta, 0))
- Initial average gain = SMA of first N gains
- Initial average loss = SMA of first N losses
- Subsequent average gain = ((previous_avg_gain * (N - 1)) + current_gain) / N
- Subsequent average loss = ((previous_avg_loss * (N - 1)) + current_loss) / N
- RS = average_gain / average_loss
- RSI = 100 - (100 / (1 + RS))
- If average loss is 0, RSI should be 100
- If average gain is 0 and average loss is 0, RSI should be 50
- Return null for candles where RSI cannot yet be calculated
- Include unit tests comparing known RSI examples

Advanced RSI indicator functions:
1. calculate_rsi(candles, period=14, source="close")
2. calculate_rsi_ma(rsi_values, period=14, ma_type="sma|ema|wma")
3. calculate_stoch_rsi(rsi_values, period=14, smooth_k=3, smooth_d=3)
4. calculate_rsi_bollinger_bands(rsi_values, period=20, std_dev=2)
5. detect_rsi_divergence(candles, rsi_values)
6. calculate_multi_timeframe_rsi(symbol, intervals=["5m","15m","1h","4h"], period=14)

Important backend behavior:
- Maintain one upstream exchange WebSocket per active symbol/interval where possible.
- Fan out normalized candle updates to multiple frontend clients.
- Avoid each frontend client opening its own exchange connection.
- Reconnect upstream exchange sockets with exponential backoff.
- Send heartbeat/ping messages to frontend clients.
- Validate all incoming WebSocket messages with Pydantic.
- Log connection events, subscription changes, upstream disconnects, and invalid messages.
- Never crash the whole server because one exchange stream fails.

Data normalization:
All exchange candle data must be converted to:
{
  time: unix_seconds,
  open: float,
  high: float,
  low: float,
  close: float,
  volume: float,
  is_closed: boolean
}

Frontend chart behavior:
- Use setData only for initial historical candles.
- Use update for live candle updates.
- Do not rerender the entire chart on every tick.
- Keep chart rendering performant.
- Debounce resize events.
- Preserve visible time range when indicator settings change.
